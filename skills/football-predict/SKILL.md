# 足球博彩预测系统

三大工作流：数据采集 → 预测分析 → 赛后复盘与在线学习
版本：v2.3

### 输出格式

分析过程→完整推理不省略 | 推送结果→精简标准格式 | **每场赛后→强制复盘**

**分析过程（内部完整版）：** 每步不省略，包含完整公式推导、多因子λ运算、蒙特卡洛模拟、异常信号7项检测、欧亚转换校验、EV计算、三路交叉验证。

**推送格式（精简版）：** 盘口+推荐+核心逻辑+波胆Dutching

**赛后复盘（强制四步法）：** 偏差分析→异常信号复盘→参数在线更新→累计准确率追踪

### 核心架构

```

        ↓
   ┌────┴────┐
   │ 亚盘模型 │         │ 大小球模型 │
   │ (独立)   │  ←→     │ (独立)     │
   └────┬────┘         └────┬────┘
        │                   │
        └──────┬────────────┘
```

## 工作流二：预测分析

系统采用**三路独立运算 + 交叉验证**架构：亚盘模型、大小球模型、波胆筛分各自独立运算，结果互相验证。单维度判断失效的地方，交叉验证能补回来。所有赔率以皇冠（Crow*）即盘为准。

---

### 一：亚盘预测模型

#### 1.1 泊松基础模型

构建零堆积双变量泊松（ZIBP）计算两队进球概率：

```
P(X=x, Y=y) = π × I(x=0,y=0) + (1-π) × P_poisson(x,y)
```

λ 参数估计采用多源高维融合：
```
λ_home = 近期主场场均进球×α + 历史交锋主场场均进球×β + xG加权×γ +
         进攻提速因子(Pace)×δ + 进攻转换率(Conversion)×ε + 关键球员权重×η
```

| 因子 | 符号 | 默认值 | 说明 |
|------|------|--------|------|
| 近期进球 | α | 0.35 | 近5/10场场均进球，含时间衰减 |
| 历史交锋 | β | 0.15 | 同主客历史交锋数据 |
| xG 加权 | γ | 0.25 | 赛季+近5场 xG/xGA |
| 进攻提速 (Pace) | δ | 0.10 | 射门频次、推进速度 |
| 进攻转换率 (Conversion) | ε | 0.05 | 射门转化进球效率 |
| 关键球员 (伤停) | η | 0.10 | 评分>7.0球员缺阵时系数 |

代入时间衰减和赛程疲劳因子后，跑 **10,000~20,000 次蒙特卡洛模拟**。

#### 1.2 得到 AI 理论盘口

蒙特卡洛输出的各比分概率 → 转为胜平负概率 → 反推理论盘口。

#### 1.3 亚盘特征加权

| 特征 | 权重 | 说明 |
|------|------|------|
| 赔率隐含概率 | **35%** | 多家主流机构赔率反推的市场共识 |
| 基本面分析 | **20%** | 近期状态、防守强度、主客场差异 |
| 球队历史数据 | **20%** | 历史交锋、相同盘口历史表现 |
| 阵容影响 | **15%** | 关键人员伤停权重（致命变量） |
| 战意因子 | **10%** | 联赛积分需求、杯赛淘汰压力 |

输出：AI理论盘口（让球数）+ 各盘口线赢/走/输概率 + 置信区间

#### 1.4 盘口对冲与庄家意图识别

- AI理论盘口 ↔ 主流机构即时盘口 **对冲**，评估盘口是否合理反映双方实力差
- 分析赔率/盘口变化趋势（早盘→即盘），识别庄家操盘意图
- **欧赔转亚盘的合理性校验**：欧赔隐含概率 → 理论亚盘 → 对比实际亚盘，偏差大即诱盘信号
- 结合主流机构资金流监控，资金流向 vs 盘口变动方向不一致 → 标记诱盘

#### 1.5 亚盘推荐

- 只推荐 **皇冠赔率 ≥ 1.75** 的亚盘方向
- 输出格式：`西班牙 -0.75 @1.85（皇冠，赢63.3% / 输27.0%）`

### 二：大小球预测模型（独立运算）

#### 2.1 多因子整合

| 因子 | 权重 | 说明 |
|------|------|------|
| **xG/xGA 预期进球数** | 核心 | 两队赛季 + 近5场 xG（进攻）/ xGA（防守），含时间衰减 |
| **进攻提速 (Pace)** | 高 | 射门频次、攻防转换速度，快节奏→大球概率↑ |
| **进攻转换率 (Conversion)** | 中 | 射门转化进球效率，高效→大球概率↑ |
| **关键人员伤停权重** | **致命变量** | 核心射手缺阵→λ下调；核心后卫缺阵→λ上调 |
| **联赛因子** | 基础 | 联赛画像（场均进球、大球率） |
| **半场进球分布模式** | 中 | 上半场/下半场进球比例，慢热/快热球队识别 |
| **角球密度** | 低 | 场均角球数，角球进攻效率 |
| **主客场环境差异** | 中 | 主场进球优势系数、客场防守下滑系数 |
| 赛程疲劳因子 | 中 | 密集赛程→防守质量↓→大球概率↑ |
| 裁判执法风格 | 低 | 黄牌/犯规越多→比赛越碎→进球↓ |

#### 2.2 AI 理论进球线

基于以上因子独立估算 λ_home、λ_away → 跑 **10,000 次以上蒙特卡洛模拟** → 输出：

- **AI理论进球线（Theoretical Line）**
- **进球期望均值**（精确到大/小/走概率分布）

#### 2.3 错价比对（筛选核心）

- AI理论进球线 vs 机构初盘/即时盘（皇冠、澳门、Bet365）
- 偏差 ≥ **0.25球** → 标记错价机会
- 结合主流机构即时盘口临场监控，捕捉变盘时机

#### 2.4 大小球推荐

- 只推荐 **皇冠赔率 ≥ 1.75** 的大小球方向
- 输出格式：`大球 2.5/3 @1.88（皇冠，55.1%）— 进3球赢半，≥4球全赢`

### 三：波胆筛选与 Dutching 对冲

#### 3.1 寻找 xG 异常值

- 分析两队近期防守/进攻核心复出/缺阵的 xG 偏差
- |实际xG - 预测xG| > 0.5 时触发加权偏移

#### 3.2 筛选 3 个最具价值波胆

- 取概率最高的 10 个比分
- 结合 xG 异常值筛选方向偏移
- 筛选出 **3 个 EV 为正**的波胆

#### 3.3 Dutching（反推对冲）计算

```
本金比例_i = (1/odds_i) / Σ(1/odds_j)
投资回报 = 总本金 / Σ(1/odds_j)
```

输出表格（赔率取皇冠即时波胆赔率）：

```
| 比分 | 概率 | 赔率 | 本金 | 返还 |
|------|:---:|:---:|:---:|:---:|
| 1-0  | 12% | 6.5  | 42  | 273 |
| 2-1  | 10% | 8.0  | 34  | 272 |
| 1-1  |  9% | 7.0  | 24  | 168 |

- 总投入：100
- 任意命中返还：XXX（回报率 X%）
- 三选覆盖概率：XX%
```

### 四：三路交叉验证（最终推荐前强制）

| 验证项 | 检查内容 | 矛盾标记 |
|--------|---------|---------|
| 亚盘 ↔ 大小球 | 亚盘预测大胜但大小球预测小球？ | ⚠️ 矛盾，降低置信度 |
| 亚盘 ↔ 波胆 | 亚盘方向与波胆池分布一致？ | ⚠️ 不一致时提醒 |
| 大小球 ↔ 波胆 | 大小球预测 vs 波胆总进球范围一致？ | ⚠️ 不一致时提醒 |
| 三路一致 | 三者方向一致 | ✅ 高置信度推荐 |

### 五：EV 计算（全部方向）

**每个投注方向含全部可投选项，全部计算 EV：**

```
EV_home  = P(home_win)  × home_odds  - P(not_home_win)
EV_draw  = P(draw)      × draw_odds  - P(not_draw)
EV_away  = P(away_win)  × away_odds  - P(not_away_win)
EV_over  = P(over)      × over_odds  - P(not_over)
EV_under = P(under)     × under_odds - P(not_under)

# 亚盘（含走水）
EV_asian_home = P(主赢)×home_water - P(主打平)×1 - P(主打输)×(1+负水)
# 走水净收益=0
```

**规则：只推荐 EV > 0 的方向。所有 EV ≤ 0 的方向不列入推荐。**

### 错价比对汇总（筛选核心）

| 维度 | 对比对象 | 阈值 |
|------|---------|------|
| 亚盘错价 | AI理论盘口 vs 机构即时盘（皇冠/澳门/Bet365） | ≥ 0.25球 |
| 大小球错价 | AI理论进球线 vs 机构大小球线 | ≥ 0.25球 |
| 欧亚转换校验 | 欧赔→理论亚盘 vs 实际亚盘 | 偏差大=诱盘 |
| 交叉机构离散度 | 皇冠/威廉/Bet365/澳门同一选项赔率 | 离散 > 5% |
| 资金流验证 | 资金流向 vs 盘口变动方向 | >60%反向→诱盘；>70%不动→强诱盘 |
| 盘口走势 | 早盘→即盘变化趋势 | 结合资金流和降价一致性 |

## ⚠️ 推荐输出规则（强制，每次预测必须遵守）

### 赔率基准
所有最终推荐的赔率，以**皇冠（Crow*）即时赔率**为唯一基准。捷报盘口页中 `Crow*` 行为基准行。皇冠即盘行读取以下字段：
- 让球即时：盘口 + 主队水位、客队水位
- 标准即时：主胜赔率、和局赔率、客胜赔率
- 进球数即时：大小球盘口 + 大球水位、小球水位

### 过滤规则
| 类型 | 规则 |
|------|------|
| 亚盘 | 皇冠即时赔率 < 1.75 → 直接过滤，不予推荐 |
| 大小球 | 皇冠即时赔率 < 1.75 → 直接过滤，不予推荐 |
| 欧赔（胜平负） | 模型 EV > 0 即推荐，不限赔率门槛 |
| 欧赔 < 1.80 | 可推荐，但必须附注 `⚠️ 赔率偏低，投注须谨慎` |

### 输出格式
1. **亚盘**：`方向 + 盘口线 + 皇冠水位 + 赢/走/输概率`
   例：`西班牙 -0.75 @1.85（皇冠，赢63.3% / 输27.0%）`
   🚨 **赔率门槛：皇冠即时赔率 ≥ 1.75 方予推荐**
   ❌ 禁止省略"皇冠"标注、禁止只写盘口不写赔率

2. **大小球**：`方向 + 盘口线 + 皇冠水位 + 概率 + 赢半/赢全说明`
   例：`大球 2.5/3 @1.88（皇冠，55.1%）— 进3球赢半，≥4球全赢`
   🚨 **赔率门槛：皇冠即时赔率 ≥ 1.75 方予推荐**
   ❌ 禁止省略盘口线、禁止省略赢半/赢全说明

3. **欧赔**：`选项 + 皇冠赔率 + 概率 + EV`
   例：`西班牙胜 @1.60（皇冠，73.5%）EV +0.176`
   若赔率 < 1.80，追加：`⚠️ 赔率偏低，投注须谨慎，建议搭配其他选项`

4. **波胆 Dutching**：使用表格格式，赔率取皇冠即时波胆赔率，含本金分配+返还+回报率+覆盖概率

### 推荐排序
- 先按 EV 降序排列
- 亚盘/大小球赔率 < 1.75 的自动过滤不出现在列表中
- 欧赔正常排序，< 1.80 的标谨慎但保留
   格式：`西班牙胜 @1.46（73.5%，澳*）EV +0.073`
   欧赔按模型 EV 正常推荐，不限赔率门槛
   ⚠️ 若赔率 < 1.80，必须在推荐后附注：`⚠️ 赔率偏低，投注须谨慎，建议搭配其他选项`

4. **波胆 Dutching** 必须用表格：比分 | 概率 | 赔率 | 本金 | 返还
   + 总投入 + 确保返佣 + 回报率 + 覆盖概率

**通用原则**：每个推荐必须零歧义、可独立操作、含完整参数、有盈利空间。

### 步骤 6：波胆筛选与 Dutching

1. 取概率最高的 10 个比分
2. 基于 xG 异常值筛选：当某队核心射手回归、xG 预测偏差 > 0.5 时，波胆池向该方向加权偏移
3. 从剩余波胆中筛选 3 个最具价值（EV > 0）的波胆
4. Dutching 本金分配：
   ```
   本金比例_i = (1/odds_i) / Σ(1/odds_j)
   投资回报 = 总本金 / Σ(1/odds_j)
   ```
5. 使用以下表格格式输出 Dutching 波胆（必须含赔率+本金+返还明细）：

```
| 比分 | 概率 | 赔率 | 本金 | 返还 |

| X-X | XX% | X.X | XX | XXX |
| X-X | XX% | X.X | XX | XXX |
| X-X | XX% | X.X | XX | XXX |

- 总投入：100
- 任意命中返还：XXX（回报率 X%）
- 三选覆盖概率：XX%
```

## 工作流三：赛后复盘（系统自动运行，四步流程）

赛后复盘流程自动触发，无需手动请求。严格执行以下四步：

---

### Step 1：偏差分析

对比预测值与实际值，量化每个维度的偏差幅度：

| 对比项 | 预测值 | 实际值 | 偏差 |
|--------|--------|--------|------|
| 比分 | - | - | - |
| 亚盘结果 | - | - | - |
| 大小球结果 | - | - | - |
| xG 偏差（主队） | - | - | - |
| xG 偏差（客队） | - | - | - |

**输出：** 量化偏差数值 + 方向对错标记（✅/❌/⏸️）

---

### Step 2：异常信号复盘

回顾赛前触发的所有异常信号，逐条对比实际比赛进程：

| 赛前异常信号 | 触发值 | 实际结果 | 判定 |
|-------------|--------|---------|------|
| 盘口变动 ≥ 0.25球 | - | - | ✅ 正确信号 / ❌ 虚警 |
| 资金流异常 | - | - | ✅ 正确 / ❌ 虚警 / ⚠️ 未验证 |
| 核心球员缺阵 | - | - | ✅ 正确 / ❌ 虚警 |
| xG 偏差 > 0.5 | - | - | ✅ 正确 / ❌ 虚警 |
| 赔率离散 > 5% | - | - | ✅ 正确 / ❌ 虚警 |

**输出：** 各信号判定 + 分析虚警率，为后续阈值微调提供依据

---

### Step 3：参数在线更新

基于 Step 1的偏差和 Step 2的信号判定，对以下参数执行增量更新：

```
W_new = W_old + η × prediction_error × W_old × anomaly_flag
```
η = 0.05（学习率），anomaly_flag = 0 或 1

**更新对象清单：**

| 参数 | 符号 | 更新触发条件 |
|------|------|------------|
| **零堆积系数** | π | 0-0比分预测偏差大时调整 |
| **时间衰减系数** | λ_decay | 近期/远期数据权重失衡时调整 |
| **泊松λ权重** | α, β, γ, δ, ε, η | 某因子高估/低估时对应调整 |
| **赛程疲劳敏感度** | δ_fatigue | 密集赛程预测偏差大时调整 |
| 联赛场均进球 | league_avg | 该联赛连续3场偏差 > 0.5球 |
| 联赛大球率 | league_over25 | 同上 |
| 错价阈值 | 0.25球 | 连续虚警则放宽，连续漏报则收紧 |
| 资金流阈值 | 60%/70% | 信号误判率高时调整 |
| 亚盘特征权重 | 35%/20%/20%/15%/10% | 某特征持续失效时调低 |

**输出：** 变更记录（参数名 + 旧值 → 新值 + 变更原因）

---

### Step 4：累计准确率追踪

更新 `memory/football-accuracy-log.md`：
- 总场次
- **亚盘准确率**（方向/盘口分开统计）
- **大小球准确率**
- 波胆命中率
- Dutching回报率
- 按联赛拆分分析

**目标：**
- 🎯 亚盘 ≥ 70%
- 🎯 大小球 ≥ 70%

同时更新 `memory/football-match-history.md` 写入本场完整记录。

**下次预测时自动读取这些记忆文件**，带着历史参数和准确率经验跑本场分析。

## 核心模型参数

### 时间衰减权重

```
W(t) = W₀ × e^(-λ × t)
```

| 数据类型 | λ |
|---------|---|
| 即时盘口 | 0.01 |
| 初盘盘口(早) | 0.10 |
| 伤停信息 | 0.03 |
| xG 基础数据 | 0.05 |
| 资金流数据 | 0.02 |

### 赛程疲劳因子

```
F = 1 + (days_since_last / 7 - 1) × δ
```

| 间隔 | F |
|------|---|
| ≤3天（一周双赛） | 0.85~0.92 |
| 7天 | 1.0 |
| >7天 | 1.02~1.05 |

应用：λ_adjusted = λ_base × F，特别影响下半场进球概率。密集赛程时大小球下调 0.15~0.25 球。

### 异常阈值

满足任一即触发：
1. |实际xG - 预测xG| > 0.5
2. 临场1h内盘口变动 ≥ 0.25球
3. 某方资金流入 > 60% 但盘口/赔率反向变动 → 诱盘提示
4. 某方资金流入 > 70% 但盘口未相应变动 → 强烈诱盘信号
5. 核心球员（评分>7.0）突发缺阵
6. 预测概率与实际概率偏离 > 1.5σ
7. 多家机构同一选项赔率离散度 > 5% → 异常标记

## 数据源

### 主数据源

捷报比分网 `nowscore.com`，依赖 browser 工具。

### 辅助数据源

**Tier 1（核心）：** Opta/StatsBomb xG、FBref/Understat xG、Betfair Exchange、Bet365/威廉希尔赔率
**Tier 2（重要）：** OddsPortal/FlashScore盘口变动、Transfermarkt/WhoScored伤停、ClubElo评分、Soccerway交锋
**Tier 3（参考）：** Weather.com天气、FIFA/UEFA排名、官方新闻

### 辅助数据整合

```
综合xG = Opta_xG×0.35 + Understat_xG×0.30 + FBref_xG×0.25 + 模型_xG×0.10
资金流评分 = Betfair×0.50 + OddsPortal成交量×0.30 + Bet365大额注码×0.20
阵容完整度 = 首发11人评分均值 / 理论最强阵容评分均值
```
